# Мехрубон & Исмат — свадебное приглашение 💍

Статический сайт-приглашение на свадьбу. Без сборки и зависимостей — чистый HTML/CSS/JS.

## Возможности
- Главное фото с плавным hover-переходом `couple_1 → couple_2`
- Чёрная тема, акцент — зелёный «плюмбоб» из Sims
- Живой обратный отсчёт до даты свадьбы
- Галерея 3×N с центрированием неполного ряда
- Форма RSVP (ответы сохраняются в `localStorage`)
- Stewie Griffin в футере 😄

## Локальный запуск
```bash
python3 -m http.server 8000
# открыть http://localhost:8000
```

## Что отредактировать перед публикацией
- **Дата свадьбы** — `script.js`, константа `WEDDING_DATE`
  ```js
  const WEDDING_DATE = new Date(2026, 8, 12, 16, 0); // год, месяц(0–11), день, час, минута
  ```
- **Место / адрес** — секция «Детали» в `index.html`
- **Сбор ответов RSVP** — по умолчанию ответы пишутся только в браузер.
  Для приёма ответов подключите Formspree / Google Forms в `script.js`.

## Деплой на GitHub Pages

### Вариант A — GitHub Actions (рекомендуется, уже настроен)
1. Создайте репозиторий на GitHub и запушьте код в ветку `main`.
2. В репозитории: **Settings → Pages → Build and deployment → Source: GitHub Actions**.
3. Каждый пуш в `main` автоматически публикует сайт (workflow `.github/workflows/deploy.yml`).

### Вариант B — Deploy from a branch
1. Запушьте код в `main`.
2. **Settings → Pages → Source: Deploy from a branch → Branch: `main` / `(root)`**.

Файл `.nojekyll` отключает обработку Jekyll, чтобы все файлы отдавались как есть.

Сайт будет доступен по адресу `https://<username>.github.io/<repo>/`.
Все пути в проекте относительные, поэтому работа из подкаталога репозитория поддерживается.
